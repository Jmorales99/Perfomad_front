## Architecture

Performad is a multi-platform ad management SaaS. The frontend follows clean architecture with three main layers:

```
src/
├── app/              # Providers + router (application shell)
├── domain/           # Entity types and models
├── infrastructure/   # API clients, Supabase, repositories
├── interface/        # Pages, layouts, hooks, UI components
├── components/       # Shared UI: Sidebar, TopHeader, shadcn/ui primitives
└── lib/              # Utility functions (cn(), error messages)
```

### Auth & State

Two global React contexts drive the app:

**AuthProvider** (`src/app/providers/AuthProvider.tsx`) — session, user profile, subscription status. Initialized by calling `supabase.auth.getSession()` on mount and subscribing to `onAuthStateChange`. The Axios interceptor reads the Supabase token on every request and auto-signs-out on 401.

**ClientProvider** (`src/app/providers/ClientProvider.tsx`) — multi-brand workspace. The selected brand (`selectedClientId`) is persisted in localStorage. Most API calls accept an optional `clientId`; without it, the backend returns account-level data.

### Routing

Routes are defined in `src/app/router/index.tsx`. Two guards wrap routes:
- `ProtectedRoute` — redirects to `/auth` if not authenticated
- `PublicRoute` — redirects to `/home` if already authenticated

The protected subtree lives under `/home` and is wrapped with `ProtectedLayout` (Sidebar + TopHeader).

### API Layer

`src/infrastructure/api/client.ts` — single Axios instance with:
- `Authorization: Bearer <supabase_token>` injected on every request
- 401 → auto sign-out + redirect to `/auth`
- All errors normalized to `ApiError` (see `errors.ts`)

Repositories are thin wrappers over the Axios client, one file per domain:

| File | Responsibility |
|---|---|
| `authRepository.ts` | Login / register via backend (not direct Supabase) |
| `profileRepository.ts` | GET/PATCH user profile |
| `campaignsRepository.ts` | Campaign CRUD, metrics, sync |
| `platformRepository.ts` | Per-platform metrics and insights |
| `optimizationRepository.ts` | AI recommendation lifecycle |
| `subscriptionRepository.ts` | OAuth connection links, connected accounts |
| `dashboardRepository.ts` | Consolidated dashboard (cached + sync) |
| `repositories/clientsRepository.ts` | Brand (client) CRUD |
| `repositories/integrations/*` | Meta / Google Ads / TikTok per-brand flows |

### Data Fetching Pattern

Pages fetch data directly in `useEffect` — no global data-fetching library. The standard shape:

```tsx
const [data, setData] = useState<T | null>(null)
const [loading, setLoading] = useState(true)
const [error, setError] = useState<string | null>(null)

useEffect(() => {
  if (!selectedClientId) return
  ;(async () => {
    setLoading(true)
    try {
      setData(await someRepository.getData(selectedClientId))
    } catch (err) {
      setError(err instanceof ApiError ? err.serverMessage ?? err.message : 'Unknown error')
    } finally {
      setLoading(false)
    }
  })()
}, [selectedClientId])
```

### Subscription Gating

`useSubscriptionGate()` (`src/interface/hooks/useSubscriptionGate.tsx`) returns `{ canAct, openPaywall, PaywallModal }`. Pages destructure this and render `<PaywallModal />` inline; actions that require a subscription call `openPaywall()` instead of proceeding.

### Testing Conventions

- Environment: happy-dom via Vitest
- Mocks: `vi.mock(...)` at module level; context values built with factory helpers
- API tests mock `@/infrastructure/api/client` directly (`apiClient.get/post/patch/delete`)
- Component tests use `@testing-library/react` + `@testing-library/jest-dom`
- Test files live alongside the code they test in `__tests__/` subdirectories

### Platform Support

Supports four ad platforms: **Meta**, **Google Ads**, **LinkedIn**, **TikTok**. TikTok connectivity is per-brand (clientId required); Meta and Google Ads are account-level. OAuth connection links are generated by `subscriptionRepository.createConnectionLink(platform, redirectUri, state)`.

### UI Stack

- **Tailwind CSS 4** — utility classes, `cn()` helper from `src/lib/utils.ts`
- **shadcn/ui** components in `src/components/ui/` (Button, Card, Dialog, Tabs, etc.)
- **Recharts** — charts and sparklines
- **Framer Motion** — page transitions and interactive animations
- **Lucide React** — icons
